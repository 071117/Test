# Security Policy

## Reporting a Vulnerability

Этот проект является учебным и не содержит конфиденциальных данных.
Если вы обнаружили проблему с безопасностью, пожалуйста, сообщите об этом:

Email: mamaevvgamzatt@gmail.com

Мы рассмотрим ваш отчет в течение 7 дней.

## Vulnerability Response Procedure

1. При получении отчета создается issue с меткой `security`
2. Проводится анализ уязвимости с использованием:
   - `bandit` для анализа кода
   - `safety` для проверки зависимостей
3. Если подтверждена критическая уязвимость:
   - Создается патч в отдельной ветке
   - Проводится тестирование изменений
   - Выполняется публичное уведомление в README.md
